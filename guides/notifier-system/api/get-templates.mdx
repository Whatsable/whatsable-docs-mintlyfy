# Get Templates API Documentation

<div className="bg-blue-50 p-6 rounded-lg my-6">
  <p className="text-lg">
    The Get Templates API allows users to retrieve a list of all templates associated with their account. Templates are pre-defined message formats that can be used for various communication purposes such as marketing alerts, utility messages, and delivery notifications.
  </p>
</div>

---

## Endpoint

<div className="bg-white p-6 rounded-lg border border-gray-200 shadow-sm">

```
GET https://api.insightssystem.com/api:AFRA_QCy/get_templates
```

</div>

---

## Authentication

<div className="bg-white p-6 rounded-lg border border-gray-200 shadow-sm">
Authentication is required for this endpoint. Include your API key in the request headers using Bearer authentication.

### Headers

| Header        | Value               | Description                              |
| ------------- | ------------------- | ---------------------------------------- |
| Content-Type  | application/json    | Specifies the format of the request body |
| Authorization | Bearer YOUR_API_KEY | Your API key prefixed with "Bearer "     |

</div>

---

## Request Parameters

<div className="bg-white p-6 rounded-lg border border-gray-200 shadow-sm">
This endpoint does not require any request parameters. All templates associated with the authenticated user will be returned.
</div>

---

## Response

<div className="bg-white p-6 rounded-lg border border-gray-200 shadow-sm">

### Response Format

The response is a JSON array containing template objects.

### Response Fields

Each template object contains the following fields:

| Field Name       | Type    | Description                                                    |
| ---------------- | ------- | -------------------------------------------------------------- |
| id               | Integer | The unique identifier for the template                         |
| user_id          | String  | UUID of the user who owns the template                         |
| name             | String  | Descriptive name of the template                               |
| category         | String  | Category of the template (e.g., "MARKETING", "UTILITY")        |
| type             | String  | The type of template (e.g., "text")                            |
| language         | String  | Language code of the template (e.g., "en", "en_US")            |
| status           | String  | Approval status of the template (e.g., "APPROVED", "REJECTED") |
| variable_counts  | Integer | Number of variable placeholders in the template                |
| template_formate | String  | Format string showing variable placement (e.g., "[b:7]")       |
| template_id      | String  | UUID of the template                                           |

### Response Codes

| Status Code | Description                                           |
| ----------- | ----------------------------------------------------- |
| 200         | Success                                               |
| 401         | Unauthorized - Invalid or missing API key             |
| 403         | Forbidden - Account does not have access to templates |
| 500         | Server Error                                          |

</div>

## Notes

<div className="bg-white p-6 rounded-lg border border-gray-200 shadow-sm">
- The `status` field indicates whether the template has been approved for use.
- The `template_formate` field shows the variable placeholders in the template. For example, "[b:7]" indicates there are 7 variables.
- Templates are returned in descending order by ID.
</div>
